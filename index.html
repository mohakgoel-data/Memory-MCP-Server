Core MCP Tools Exposed
💾
remember()
Write memory
Input
Raw conversation text or explicit fact
Process
LLM extracts structured facts, tags category, assigns strength score 1.0
Output
Stored memory entry with timestamp + embedding vector
🔍
recall()
Read memory
Input
Current query or conversation context
Process
Semantic similarity search, filters by decay score, ranks by relevance
Output
Top-N relevant memories injected as context, strength score boosted
🗑️
forget()
Delete memory
Input
Memory ID or natural language description
Process
Finds matching memory entry, hard deletes or decay-zeroes it
Output
Confirmed deletion — user stays in full control of stored data
Memory Schema & Recall Flow
📋
Memory Entry Schema
How each memory is stored
// memory_entry
{
  "id": uuid,
  "user_id": string,
  "content": "prefers Python over JS",
  "category": "preference",
  // preference | project | decision | fact
  "strength": 0.84,
  // 0.0–1.0, decays over time
  "embedding": vector(1536),
  "created_at": timestamp,
  "last_recalled": timestamp
}
🔄
Recall Flow — Step by Step
What happens on every new session
1
Agent calls recall() with current user query as context
recall({ query: "help me build a React app" })
2
Query is embedded, semantic search run against memory store
pgvector cosine similarity → top 5 results
3
Results filtered by decay score — weak/old memories excluded
WHERE strength > 0.3 ORDER BY relevance DESC
4
Memories injected into agent's system prompt automatically
→ agent already knows user's stack, preferences, projects

What Makes This Novel
Protocol-level, not product-level
Memory lives outside any single AI tool. Any MCP-compatible agent plugs in with zero custom integration code.
Memory decay mechanism
Memories have a strength score that weakens over time unless reinforced — mimicking how human memory actually works.
Full user control — Memory Vault
Users see, edit, and delete exactly what is stored. Transparent memory builds trust. Most solutions don't offer this.
